## Architecture

This is a **Next.js 16 App Router** site that renders population data stored in an **Umbraco headless CMS** via the Umbraco Delivery API.

### Data flow

```
Umbraco CMS ‚Üí Delivery API ‚Üí orval-generated client (src/api/) ‚Üí umbraco/index.ts ‚Üí pages
```

**`src/api/`** is fully generated by orval ‚Äî never edit it by hand. Run `npm run generate` after any Umbraco content model change. orval reads the OpenAPI spec from the live Umbraco instance and writes typed models to `src/api/model/` and fetch functions to `src/api/content/` and `src/api/media/`.

**`src/custom-fetch.ts`** is the orval fetch mutator. It rewrites the placeholder base URL in the generated client to `NEXT_PUBLIC_UMBRACO_BASE_URL`, handles empty/non-JSON responses safely, and normalises the return shape to `{ status, data, headers }`.

**`src/umbraco/index.ts`** is the only place that calls the generated API functions. All pages import from here, never directly from `src/api/`.

**`src/lib/utils.ts`** exports `toSlug(path)` which strips leading/trailing slashes from Umbraco `route.path` values (e.g. `/europe/` ‚Üí `europe`) for use in URLs and route param comparisons.

### URL structure

```
/                          ‚Üí lists all continents with their countries
/[continent]               ‚Üí continent page
/[continent]/[country]     ‚Üí country page (flag, description, population chart + table)
```

Route params are bare slugs; Umbraco `route.path` values have leading and trailing slashes ‚Äî always use `toSlug()` to bridge them.

### Umbraco content model

| Type | Properties used |
|---|---|
| `settings` | `siteName`, `disclaimer` |
| `continent` | ‚Äî |
| `country` | `capital`, `description` (RichTextModel), `iso2Code`, `iso3Code` |
| `year` | `population` |

### External data sources

- **Flags** ‚Äî served from `https://flagcdn.com` using the country's `iso2Code` property (e.g. `flagcdn.com/w80/gb.webp`). No API key required.
- **Population figures** ‚Äî sourced from the [World Bank public API](https://data.worldbank.org) and stored as `year` content nodes in Umbraco. The site does not call the World Bank API at runtime.

### Caching and revalidation

All Umbraco fetches are tagged with `"content"` and use `revalidate: 3600`. When Umbraco publishes content it POSTs to `/api/revalidate` with an HMAC-SHA256 signature verified against `UMBRACO_REVALIDATE_SECRET`.

### Styling

All styles live in `src/styles/globals.scss`. Design tokens are CSS custom properties in `:root`. The only Sass variable is `$bp-sm: 600px` ‚Äî breakpoints must stay as Sass vars because CSS custom properties cannot be used in `@media` queries.

## Environment variables

| Variable | Purpose |
|---|---|
| `NEXT_PUBLIC_UMBRACO_BASE_URL` | Umbraco base URL, no trailing slash ‚Äî used at runtime |
| `MODEL_GENERATE_BASE_URL` | Local Umbraco base URL used only during `npm run generate` |
| `UMBRACO_REVALIDATE_SECRET` | HMAC secret for the revalidation webhook |
| `UMBRACO_REVALIDATE_ACCESS_CONTROL_ORIGIN` | CORS origin for the revalidation endpoint |
| `NODE_TLS_REJECT_UNAUTHORIZED` | Set to `0` in `.env.local` for local dev with self-signed certs |
